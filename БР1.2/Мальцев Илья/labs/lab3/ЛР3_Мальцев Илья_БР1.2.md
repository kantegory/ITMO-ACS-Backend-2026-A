## Задача

Реализовать Dockerfile для каждого сервиса, написать общий `docker-compose.yml` и настроить сетевое взаимодействие между сервисами.

## Ход работы

Микросервисное приложение было разнесено по отдельным Docker-контейнерам:

- `api-gateway`;
- `auth-service`;
- `catalog-service`;
- `applicant-service`;
- `employer-service`;
- `notification-service`;
- `rabbitmq`.

Для каждого backend-сервиса создан отдельный Dockerfile в папке `deploy`.

Общий `docker-compose.yml` описывает сборку контейнеров, переменные окружения, healthcheck-и и сеть `job-search-network`.

Взаимодействие между сервисами настроено через имена контейнеров Docker Compose. Например, `api-gateway` обращается к `auth-service` по адресу `http://auth-service:8081`, а `applicant-service` публикует события в RabbitMQ по адресу `amqp://guest:guest@rabbitmq:5672/`.

## Проверка

Запуск приложения:

```bash
docker compose up --build
```

Проверка gateway:

```bash
curl http://localhost:8080/api/v1/health
curl http://localhost:8080/api/v1/vacancies
```

Проверка RabbitMQ:

```text
http://localhost:15673
guest / guest
```
