# Postman

В папке лежат готовые файлы для ДЗ3:

- `restaurant-booking-main-flow.postman_collection.json` — основной сценарий приложения
- `restaurant-booking-local.postman_environment.json` — локальное окружение

Сценарий покрывает такой поток:

1. Регистрация пользователя
2. Вход пользователя
3. Получение профиля
4. Вход администратора
5. Создание локации
6. Создание ресторана
7. Добавление столика
8. Добавление меню и фото
9. Публикация ресторана
10. Поиск ресторана
11. Просмотр карточки
12. Проверка доступности столиков
13. Создание бронирования
14. Получение истории бронирований
15. Создание отзыва
16. Подтверждение бронирования администратором

Как запускать:

1. Убедиться, что PostgreSQL и backend уже запущены
2. Импортировать оба JSON-файла в Postman
3. Выбрать environment `Restaurant Booking API Local`
4. Запустить коллекцию `Restaurant Booking API - Main Flow` через Collection Runner
5. Смотреть, чтобы все тесты были зелёными

Что важно:

- коллекция сама генерирует уникальные `email` и `phone` для нового пользователя
- `admin`-логин берётся из `.env`
- `restaurantId`, `locationId`, `tableId`, `reservationId` и другие идентификаторы сохраняются автоматически между запросами через переменные коллекции
