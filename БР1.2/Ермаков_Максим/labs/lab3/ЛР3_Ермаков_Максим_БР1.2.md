## Цель работы

Цель лабораторной работы — контейнеризировать микросервисное backend-приложение:
реализовать Dockerfile для каждого сервиса, написать общий `docker-compose.yml`
и настроить сетевое взаимодействие между контейнерами.

Работа выполнена в папке `labs/lab3`. Эта папка является отдельной версией
проекта: `lab2` остается запускаемой отдельно, а `lab3` содержит RabbitMQ из
ДЗ5 и Docker-контейнеризацию для ЛР3.

## Dockerfile

Для каждого runtime-компонента добавлен отдельный Dockerfile:

| Компонент | Dockerfile |
|---|---|
| `api-gateway` | `docker/Dockerfile.gateway` |
| `identity-service` | `docker/Dockerfile.identity` |
| `catalog-service` | `docker/Dockerfile.catalog` |
| `menu-service` | `docker/Dockerfile.menu` |
| `reservation-service` | `docker/Dockerfile.reservation` |
| `review-service` | `docker/Dockerfile.review` |

Каждый Dockerfile выполняет одинаковую схему:

1. устанавливает зависимости через `npm ci`;
2. копирует `tsconfig.json` и `src`;
3. собирает TypeScript-проект командой `npm run build`;
4. в runtime-образ копирует только production-зависимости и папку `dist`;
5. запускает нужный JS-файл конкретного сервиса.

## Docker Compose

В общий `docker-compose.yml` включены:

| Контейнер | Назначение | Порт хоста |
|---|---|---:|
| `db` | PostgreSQL с отдельными БД сервисов | `25432` |
| `rabbitmq` | RabbitMQ + Management UI | `5672`, `15672` |
| `api-gateway` | Публичный API | `8200` |
| `identity-service` | Пользователи и JWT | `8201` |
| `catalog-service` | Каталог ресторанов и consumer RabbitMQ | `8202` |
| `menu-service` | Меню ресторанов | `8203` |
| `reservation-service` | Столики и бронирования | `8204` |
| `review-service` | Отзывы и producer RabbitMQ | `8205` |

Порты `8200-8205` и `25432` выбраны специально, чтобы `lab3` не конфликтовала с
`lab2`, где используются `8100-8105` и `15432`.

## Сетевое взаимодействие

Все контейнеры находятся в одной Docker Compose network. Поэтому сервисы
обращаются друг к другу не через `127.0.0.1`, а через DNS-имена сервисов:

```text
identity-service -> http://identity-service:8201
catalog-service -> http://catalog-service:8202
menu-service -> http://menu-service:8203
reservation-service -> http://reservation-service:8204
review-service -> http://review-service:8205
db -> postgres://db:5432
rabbitmq -> amqp://rabbitmq:5672
```

Внутри контейнеров сервисы слушают `0.0.0.0`, чтобы Docker мог пробросить порт
наружу на хост.

## Запуск

Запуск всей системы:

```bash
docker compose up --build -d
```

Публичный API:

```text
http://127.0.0.1:8200/api/v1
```

RabbitMQ Management UI:

```text
http://127.0.0.1:15672
```

Логин и пароль RabbitMQ:

```text
maindb / maindb
```

## Проверка

Проверить состояние контейнеров:

```bash
docker compose ps
```

Проверить health endpoint gateway:

```bash
curl http://127.0.0.1:8200/health
```

Запустить smoke-тест с хоста:

```bash
npm run smoke
```

Smoke-тест проходит полный сценарий: создание ресторана, столика, меню, фото,
публикацию ресторана, регистрацию пользователя, бронирование и создание отзыва.
После создания отзыва тест проверяет, что рейтинг ресторана обновился через
RabbitMQ-событие из ДЗ5.
