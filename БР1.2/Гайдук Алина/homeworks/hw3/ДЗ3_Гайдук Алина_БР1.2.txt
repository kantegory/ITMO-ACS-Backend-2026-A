Задача

По заданию ДЗ3 нужно протестировать REST API приложения RecipeHub средствами Postman и оформить проверки прямо в коллекции (вкладки Tests у запросов). В результате требуется один связный сценарий, который отражает основной пользовательский процесс в сервисе, а не набор разрозненных запросов без логики.

Ход работы

Подняла backend RecipeHub из каталога labs/lab1 (Docker Compose или локальный запуск), чтобы Postman ходил на рабочий baseUrl, по умолчанию http://localhost:8080.

Подготовила коллекцию RecipeHub-DZ3.postman_collection.json и окружение RecipeHub-local.postman_environment.json с переменной baseUrl. Импортировала их в Postman и выбрала окружение RecipeHub local.

Собрала один сценарий из десяти шагов, которые выполняются по порядку в Collection Runner:

1) проверка healthz, что сервис отвечает;
2) регистрация нового пользователя с уникальными email и username через Pre-request скрипт, сохранение access_token в переменную коллекции;
3–5) получение справочников (типы блюд, сложность, ингредиенты) и сохранение id для тела следующего запроса;
6) создание рецепта от имени зарегистрированного пользователя с шагом приготовления и строкой состава;
7) запрос списка рецептов с пагинацией и проверкой полей data и pagination;
8) поиск по строке (HW3 в названии созданного рецепта), выбор id из выдачи или запасной вариант через сохранённый recipeId;
9) получение полной карточки рецепта по id;
10) отправка комментария к этому рецепту.

На каждом шаге в Tests написаны проверки через pm.test: ожидаемые коды ответа, структура JSON (наличие нужных полей, непустые массивы там, где нужно данные из справочников). Так при прогоне Runner сразу видно, на каком звене цепочки что-то сломалось.

Прогон коллекции выполняла через Run folder по папке со всеми запросами сверху вниз; при необходимости можно повторить то же из командной строки через Newman с теми же json-файлами.

Вывод

ДЗ3 закрепило навык проверять API не только «глазами» по одному запросу, а описывать типовой сценарий как воспроизводимый прогон с автоматическими assert в Postman. Связка регистрация – справочники – создание контента – список и поиск – детальная карточка – комментарий хорошо ложится на домен RecipeHub и совпадает с логикой, которую ждали по формулировке задания. Имея коллекцию и окружение в репозитории, сценарий можно повторить перед сдачей или перед релизом без ручного набора всех запросов заново.
