# Домашняя работа №1: Проектирование базы данных

## Вариант проекта: Приложение для бронирования столиков в ресторанах

## 1. Общее описание предметной области

Приложение для бронирования столиков в ресторанах предназначено для упрощения процесса резервирования столиков в различных ресторанах. Платформа позволяет пользователям находить рестораны, просматривать доступные столики и бронировать их на определенное время. Рестораторы могут управлять информацией о своем заведении, столиках и принимать бронирования.

## 2. Описание функциональных требований

### Для клиентов:
- Регистрация и аутентификация в системе
- Поиск ресторанов с фильтрацией по кухне, расположению и цене
- Просмотр информации о ресторане (меню, фото, отзывы)
- Бронирование столиков на определенное время и дату
- Просмотр истории бронирований
- Оставление отзывов о посещенных ресторанах

### Для владельцев ресторанов:
- Регистрация своего заведения в системе
- Управление информацией о ресторане
- Управление столиками (добавление, редактирование, удаление)
- Просмотр и управление бронированиями
- Просмотр отзывов о ресторане

### Для администраторов:
- Модерация контента
- Управление пользователями
- Решение спорных ситуаций

## 3. ERD-диаграмма базы данных

```
[Пользователи] ||--o{ [Бронирования] }o--|| [Столики]
[Рестораны] ||--o{ [Столики]
[Рестораны] ||--o{ [Меню]
[Пользователи] ||--o{ [Отзывы] }o--|| [Рестораны]
```

### Таблица: Пользователи (users)
- id (PK) - уникальный идентификатор пользователя
- email (unique) - электронная почта пользователя
- password_hash - хэш пароля
- first_name - имя пользователя
- last_name - фамилия пользователя
- phone_number - номер телефона
- role - роль пользователя (client, restaurant_owner, admin)
- created_at - дата создания записи
- updated_at - дата последнего обновления записи

### Таблица: Рестораны (restaurants)
- id (PK) - уникальный идентификатор ресторана
- owner_id (FK to users.id) - идентификатор владельца
- name - название ресторана
- description - описание ресторана
- address - адрес
- phone_number - номер телефона
- cuisine_type - тип кухни
- opening_time - время открытия
- closing_time - время закрытия
- rating - рейтинг ресторана
- created_at - дата создания записи
- updated_at - дата последнего обновления записи

### Таблица: Столики (tables)
- id (PK) - уникальный идентификатор столика
- restaurant_id (FK to restaurants.id) - идентификатор ресторана
- table_number - номер столика
- capacity - вместимость (количество мест)
- location_description - описание расположения
- is_available - доступность столика
- created_at - дата создания записи
- updated_at - дата последнего обновления записи

### Таблица: Бронирования (bookings)
- id (PK) - уникальный идентификатор бронирования
- user_id (FK to users.id) - идентификатор пользователя
- table_id (FK to tables.id) - идентификатор столика
- booking_date - дата бронирования
- start_time - время начала
- end_time - время окончания
- number_of_guests - количество гостей
- special_requests - специальные пожелания
- status - статус бронирования (pending, confirmed, cancelled, completed)
- created_at - дата создания записи
- updated_at - дата последнего обновления записи

### Таблица: Меню (menus)
- id (PK) - уникальный идентификатор позиции меню
- restaurant_id (FK to restaurants.id) - идентификатор ресторана
- name - название блюда
- description - описание блюда
- price - цена
- category - категория
- is_available - доступность
- created_at - дата создания записи
- updated_at - дата последнего обновления записи

### Таблица: Отзывы (reviews)
- id (PK) - уникальный идентификатор отзыва
- user_id (FK to users.id) - идентификатор пользователя
- restaurant_id (FK to restaurants.id) - идентификатор ресторана
- rating - рейтинг (1-5)
- comment - текст комментария
- created_at - дата создания записи
- updated_at - дата последнего обновления записи

## 5. Заключение

База данных спроектирована с учетом всех функциональных требований к приложению для бронирования столиков в ресторанах. Структура БД позволяет эффективно хранить и извлекать информацию о пользователях, ресторанах, столиках, бронированиях, меню и отзывах. Связи между таблицами обеспечивают целостность данных и возможность выполнения сложных запросов для реализации всех функций приложения.
