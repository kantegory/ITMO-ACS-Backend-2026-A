# Описание моделей базы данных

## users

Модель хранит информацию о пользователях системы.

| Поле | Тип | Описание |
|---|---|---|
| `user_id` | PK | Уникальный идентификатор пользователя |
| `full_name` | VARCHAR | Полное имя пользователя |
| `email` | VARCHAR | Электронная почта пользователя |
| `password_hash` | VARCHAR | Хеш пароля пользователя |
| `phone` | VARCHAR | Номер телефона пользователя |
| `phone` | VARCHAR | Номер телефона пользователя |

**Связи:**
- один пользователь может оставить много отзывов;
- один пользователь может создать много бронирований.

---

## restaurants

Модель хранит основную информацию о ресторанах.

| Поле | Тип | Описание |
|---|---|---|
| `restaurant_id` | PK | Уникальный идентификатор ресторана |
| `name` | VARCHAR | Название ресторана |
| `description` | TEXT | Описание ресторана |
| `address` | VARCHAR | Адрес ресторана |
| `city` | VARCHAR | Город, в котором находится ресторан |
| `price_category` | VARCHAR | Ценовая категория ресторана |
| `opening_time` | TIME | Время открытия ресторана |
| `closing_time` | TIME | Время закрытия ресторана |
| `created_at` | TIMESTAMP | Дата и время создания записи |

**Связи:**
- один ресторан может иметь много фотографий;
- один ресторан может иметь много меню;
- один ресторан может иметь много столиков;
- один ресторан может иметь много отзывов;
- один ресторан может иметь много бронирований;
- один ресторан может относиться к нескольким кухням через таблицу `restaurant_cuisines`.

---

## cuisines

Модель хранит список типов кухонь.

| Поле | Тип | Описание |
|---|---|---|
| `cuisine_id` | PK | Уникальный идентификатор кухни |
| `name` | VARCHAR | Название кухни |

**Связи:**
- одна кухня может быть связана с несколькими ресторанами через таблицу `restaurant_cuisines`.

---

## restaurant_cuisines

Промежуточная таблица для связи ресторанов и кухонь по принципу многие-ко-многим.

| Поле | Тип | Описание |
|---|---|---|
| `restaurant_id` | PK, FK | Идентификатор ресторана |
| `cuisine_id` | PK, FK | Идентификатор кухни |

**Связи:**
- связывает таблицу `restaurants` с таблицей `cuisines`;
- один ресторан может иметь несколько кухонь;
- одна кухня может относиться к нескольким ресторанам.

---

## restaurant_photos

Модель хранит фотографии ресторанов.

| Поле | Тип | Описание |
|---|---|---|
| `id` | PK | Уникальный идентификатор фотографии |
| `restaurant_id` | FK | Идентификатор ресторана |
| `photo_url` | VARCHAR | Ссылка на фотографию ресторана |
| `uploaded_at` | TIMESTAMP | Дата и время загрузки фотографии |

**Связи:**
- каждая фотография относится к одному ресторану;
- один ресторан может иметь много фотографий.

---

## restaurant_menus

Модель хранит файлы меню ресторанов.

| Поле | Тип | Описание |
|---|---|---|
| `id` | PK | Уникальный идентификатор меню |
| `restaurant_id` | FK | Идентификатор ресторана |
| `file_name` | VARCHAR | Название файла меню |
| `file_url` | VARCHAR | Ссылка на файл меню |
| `file_type` | VARCHAR | Тип файла меню |
| `uploaded_at` | TIMESTAMP | Дата и время загрузки файла |

**Связи:**
- каждое меню относится к одному ресторану;
- один ресторан может иметь много файлов меню.

---

## tables

Модель хранит информацию о столиках в ресторанах.

| Поле | Тип | Описание |
|---|---|---|
| `table_id` | PK | Уникальный идентификатор столика |
| `restaurant_id` | FK | Идентификатор ресторана |
| `table_number` | INT | Номер столика |
| `capacity` | INT | Количество мест за столиком |
| `location_description` | VARCHAR | Описание расположения столика |
| `is_active` | BOOLEAN | Признак доступности столика для бронирования |

**Связи:**
- каждый столик относится к одному ресторану;
- один ресторан может иметь много столиков;
- один столик может использоваться в нескольких бронированиях.

---

## reservations

Модель хранит информацию о бронированиях столиков.

| Поле | Тип | Описание |
|---|---|---|
| `reservation_id` | PK | Уникальный идентификатор бронирования |
| `user_id` | FK | Идентификатор пользователя, который сделал бронирование |
| `restaurant_id` | FK | Идентификатор ресторана |
| `table_id` | FK | Идентификатор забронированного столика |
| `reservation_datetime` | TIMESTAMP | Дата и время бронирования |
| `guest_count` | INT | Количество гостей |
| `status` | VARCHAR | Статус бронирования |
| `created_at` | TIMESTAMP | Дата и время создания бронирования |

**Связи:**
- каждое бронирование принадлежит одному пользователю;
- каждое бронирование относится к одному ресторану;
- каждое бронирование связано с одним столиком;
- один пользователь может создать много бронирований;
- один ресторан может иметь много бронирований.

---

## reviews

Модель хранит отзывы пользователей о ресторанах.

| Поле | Тип | Описание |
|---|---|---|
| `review_id` | PK | Уникальный идентификатор отзыва |
| `user_id` | FK | Идентификатор пользователя, оставившего отзыв |
| `restaurant_id` | FK | Идентификатор ресторана |
| `rating` | INT | Оценка ресторана |
| `comment` | TEXT | Текст отзыва |
| `created_at` | TIMESTAMP | Дата и время создания отзыва |

**Связи:**
- каждый отзыв принадлежит одному пользователю;
- каждый отзыв относится к одному ресторану;
- один пользователь может оставить много отзывов;
- один ресторан может иметь много отзывов.
