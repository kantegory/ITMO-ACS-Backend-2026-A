# Отчёт по лабораторной работе 1

## Тема

Реализация REST API на основе boilerplate для приложения бронирования столиков в ресторанах.

## Цель

Создать работающее REST API по варианту, используя результаты ДЗ1 и ДЗ2: описание моделей базы данных и спецификацию API.

## Использованные технологии

- Node.js
- Express
- TypeScript
- In-memory хранилище данных

## Реализованная структура

- `src/models` - TypeScript-модели предметной области.
- `src/data` - временное хранилище данных вместо реальной БД.
- `src/views` - форматирование успешных и ошибочных JSON-ответов.
- `src/controllers` - обработчики бизнес-логики.
- `src/routes` - маршруты REST API.
- `src/app.ts` - настройка Express-приложения.
- `src/index.ts` - запуск сервера.

## Реализованные модели

По описанию из `homeworks/hw1/db.md` реализованы модели: `User`, `Restaurant`, `Cuisine`, `RestaurantCuisine`, `RestaurantPhoto`, `RestaurantMenu`, `RestaurantTable`, `Reservation`, `Review`.

## Реализованные возможности

- регистрация и авторизация пользователя;
- получение и обновление профиля;
- просмотр ресторанов с фильтрацией;
- просмотр карточки ресторана, кухонь, столиков, фотографий и меню;
- создание и просмотр отзывов;
- создание, просмотр и отмена бронирований;
- базовая проверка авторизации через Bearer-токен.

## Проверка

Установка и запуск:

```sh
npm install
npm run dev
```

Проверочные запросы:

```sh
curl http://localhost:3001/
curl http://localhost:3001/api/restaurants
curl -H "Authorization: Bearer user-1" http://localhost:3001/api/users/me
```

Сборка:

```sh
npm run build
```
