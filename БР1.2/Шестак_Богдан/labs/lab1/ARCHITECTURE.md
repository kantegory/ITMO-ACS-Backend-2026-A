# Архитектура API

## Обзор

Job Search API - это REST API для платформы поиска работы. Построен на:
- **FastAPI** - современный веб-фреймворк для Python
- **SQLAlchemy** - ORM для работы с БД
- **Pydantic** - валидация данных
- **JWT** - аутентификация

## Слои архитектуры

### 1. Models (models.py)
SQLAlchemy ORM модели, описывающие структуру БД:
- `User` - пользователи (соискатели и работодатели)
- `Resume` - резюме соискателей
- `Vacancy` - вакансии компаний
- `Application` - отклики на вакансии
- `Skill` - справочник навыков
- `Company` - компании работодателей
- `WorkExperience` - опыт работы
- `Education` - образование

### 2. Schemas (schemas.py)
Pydantic модели для валидации запросов и ответов:
- Request моделях - для входящих данных
- Response модели - для исходящих данных
- Поддержка вложенных структур

### 3. Database (database.py)
Конфигурация подключения к БД:
- SQLite для разработки
- Session factory
- Dependency injection для сессий

### 4. Authentication (auth.py)
Функции безопасности:
- Хеширование пароля (bcrypt)
- Создание и проверка JWT токенов
- Access и refresh токены

### 5. Dependencies (dependencies.py)
Зависимости FastAPI:
- Извлечение текущего пользователя из токена
- Проверка прав доступа

### 6. Routers (routers_*.py)
Маршруты (эндпоинты) разделены по модулям:
- `routers_auth.py` - аутентификация
- `routers_users.py` - профиль пользователя
- `routers_vacancies.py` - вакансии
- `routers_resumes.py` - резюме
- `routers_applications.py` - отклики
- `routers_skills.py` - навыки

### 7. Main (main.py)
Инициализация приложения:
- Создание таблиц БД
- Подключение маршрутов
- CORS конфигурация
- Health check эндпоинт

## Flow диаграмма

```
Request → CORS Middleware → Route → Dependency → Handler → Response
           ↓
        Validation (Pydantic)
           ↓
        Database (SQLAlchemy)
           ↓
        Auth (JWT)
```

## Аутентификация

### Регистрация
```
POST /auth/register
└── Создать пользователя
└── Если работодатель → Создать компанию
└── Выдать access и refresh токены
```

### Вход
```
POST /auth/login
└── Проверить email и пароль
└── Выдать access и refresh токены
```

### Защита эндпоинтов
```
GET /resumes (Protected)
└── Извлечь токен из заголовка Authorization
└── Проверить токен и получить user_id
└── Выполнить операцию от имени пользователя
```

## Авторизация

### На основе роли
- `applicant` - может создавать резюме, искать вакансии, откликаться
- `employer` - может создавать вакансии, просматривать отклики

### На основе владения
- Пользователь может редактировать только свои резюме
- Работодатель может управлять только своими вакансиями

## Примеры использования

### 1. Простой поиск
```python
GET /vacancies?city=San Francisco&salary_from=100000
```

### 2. Создание резюме с опытом
```
POST /resumes
POST /resumes/{resume_id}/experience
POST /resumes/{resume_id}/education
PUT /resumes/{resume_id}/skills
```

### 3. Управление вакансией
```
POST /vacancies (создать)
GET /vacancies/my (мои вакансии)
GET /vacancies/{id}/applications (отклики)
PATCH /applications/{id}/status (изменить статус)
```

## Error Handling

Стандартные HTTP коды ошибок:
- `200` - успешно
- `201` - создано
- `204` - успешно (без контента)
- `400` - неверный запрос
- `401` - не аутентифицирован
- `403` - доступ запрещен
- `404` - не найдено
- `409` - конфликт (например, дублирование)
- `422` - неверные данные

Все ошибки возвращают JSON:
```json
{
  "detail": "Error message"
}
```
