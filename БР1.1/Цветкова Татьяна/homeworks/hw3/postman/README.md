# Postman-тесты (ДЗ3)

Комплексный E2E-сценарий для REST API «Платформа фитнес-тренировок и здоровья».

## Файлы

- `FitnessPlatform.postman_collection.json` — коллекция запросов и тестов
- `FitnessPlatform.postman_environment.json` — окружение (переменные)

## Импорт в Postman

1. В Postman: `Import` (левый верхний угол).
2. Перетащи оба файла или выбери их через диалог.
3. В правом верхнем углу выбери окружение **«Fitness Platform — local»**.

## Запуск

### Перед запуском поднять API

В терминале из корня проекта:

```bash
npm run dev
```

(Сервер должен слушать `http://localhost:3000`. Базу заполнить через `npm run seed`, если ещё не делала.)

### Запуск всех тестов одной кнопкой

1. В Postman кликни на коллекцию **Fitness Platform API — E2E сценарий**.
2. Нажми **Run** (синяя кнопка справа сверху).
3. В открывшейся вкладке Collection Runner нажми **Run Fitness Platform API**.
4. Дождись отчёта — должны пройти **все 50+ assertion'ов**.

### Запуск отдельных шагов

Запросы пронумерованы от 1 до 18 — кликай по очереди и жми **Send**. Переменные (`accessToken`, `workoutId`, `planId`, `planItemId`) сохраняются автоматически в Tests-скриптах.

## Сценарий

1. Регистрация нового пользователя (уникальный email через timestamp)
2. Вход
3. Негативный кейс: вход с неверным паролем → 401
4. Личный кабинет: получение профиля
5. Обновление профиля (вес, рост, уровень)
6. Список всех тренировок
7. Поиск с фильтрами `type=cardio&level=beginner&maxDuration=30` (запоминает id случайной)
8. Просмотр выбранной тренировки (страница с видео)
9. Негативный кейс: несуществующая тренировка → 404
10. Создание плана тренировок
11. Добавление тренировки в план
12. Отметка элемента плана как выполненного
13. Запись прогресса о тренировке
14. Негативный кейс: невалидный рейтинг (99) → 422 с деталями валидации
15. Получение сводной статистики прогресса
16. Список постов блога
17. Негативный кейс: защищённый эндпоинт без токена → 401
18. Негативный кейс: создание тренировки обычным user → 403

## Что проверяется в тестах

- Корректные HTTP-статусы (200, 201, 401, 403, 404, 422)
- Структура и типы полей в ответах
- Сохранение и согласованность данных между шагами (один пользователь, одна тренировка, один план)
- Корректная работа фильтрации (все результаты соответствуют параметрам)
- Не утекает `passwordHash`
- Время ответа (`< 1000 мс` для логина)
- Детали ошибок валидации (поле + constraints)
- Ролевая модель (user не может создавать тренировки)
